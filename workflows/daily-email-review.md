# Daily ProtonMail Client Review

## Purpose
Daily review of ProtonMail inbox to identify client emails and ensure proper labeling/organization.

## Schedule
- **Frequency:** Daily (morning recommended)
- **Channel:** #unfergettable-designs reminder
- **Duration:** ~10-15 minutes

---

## Review Process

### 1. Scan Inbox (5 min)
- [ ] Open ProtonMail inbox
- [ ] Scan for emails from known clients (see client list below)
- [ ] Note any new senders that might be prospects/clients

### 2. Label & Organize (5-10 min)
Apply appropriate labels to each client email:

**Label Structure:**
- `Clients/[ClientName]` - Per-client labels
- `Status/Active` - Active project communication
- `Status/Lead` - Potential new client
- `Status/Completed` - Finished project threads
- `Priority/Urgent` - Needs same-day response
- `Priority/This-Week` - Response needed this week
- `Invoices` - Invoice-related
- `Contracts` - Contract/agreement threads

### 3. Action Items
For each email, decide:
- [ ] **Reply** - Respond now if quick (<2 min)
- [ ] **Schedule** - Block time for longer replies
- [ ] **Archive** - Filed, no action needed
- [ ] **Flag** - Needs follow-up

### 4. Update Tracking
- Log any new client communications in daily notes
- Update project status if needed
- Flag any urgent items to #unfergettable-designs

- `Status`

- `Contracts`

**Client Sub-labels:**
- Create `Clients/[ClientName]` for each active client

---

## Notes

- ProtonMail doesn't have API access like Gmail, so this is a manual process
- Consider using ProtonMail's "Folders" vs "Labels" based on your preference
- Labels allow multiple categories per email; folders are exclusive
- Set up filters for known client email addresses to auto-label when possible
